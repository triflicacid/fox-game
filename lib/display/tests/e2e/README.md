# Playwright Visual Tests

This folder contains browser screenshot tests for `InteractableDisplay`.

## What is here

- `interactable-display.visual.spec.ts` - screenshot tests for canonical interaction scenes.
- `../harness/index.html` and `../harness/main.ts` - a tiny browser harness that renders deterministic scenes from query-string scenarios.
- `interactable-display.visual.spec.ts-snapshots/` - Playwright baselines generated by `--update-snapshots`.

Current scenario coverage includes:

- mixed-control builder layouts
- select dropdown open/highlight states
- click-focus and pressed-state rendering
- top/centre/bottom alignment mixes across text and varied inputs
- focused background-color matrices across varied input kinds
- focused font-size change scenes across varied input kinds
- padding and margin stress scenes (especially Win98 chrome)
- disabled-overlay matrices for varied input kinds in both themes
- textbox caret/selection highlight rendering
- dropdown disabled-row visual + skip-navigation states
- HR length-resolution scenes (`max`/`top`)
- nested text-segment font scaling inside controls

Screenshot assertions are configured for strict diffing (`threshold: 0`,
`maxDiffPixels: 0`) to catch even tiny visual deltas in the canonical baseline
environment.

## Commands

From `lib/display`:

```bash
pnpm install
pnpm run playwright:install
pnpm run test:e2e
pnpm run test:e2e:update
```

## Adding a new screenshot scene

1. Add a scenario in `tests/harness/main.ts`.
2. Add the scenario key to `scenarios` in `tests/e2e/interactable-display.visual.spec.ts`.
3. Run `pnpm run test:e2e:update` to generate the new baseline.
